##关于el的debug
###解决方案：
我想你的问题应该是，后台已经将数据存入了某个域对象中后，接着在页面显示的时候用el表达式取值。 此时，想确认该表达式中的值是否存在？

####依照我以往的个人经验，分两种方式：

不借助任何工具
 借助框架（主要使用struts框架）


首先，在不借助任何工具的情况下，该怎么做呢？想想el表达式当初出现的意义，就不难想到用java脚本表达式来作为测试的另一种思路。由于el表达式是java脚本表达式的替代方案，目的是为了提高代码的可读性和开发人员的开发效率。另外，值得一提的是，jstl标签恰恰是java脚本代码的替代方案，因此，我们也可以通过写脚本代码的方式来测试。这是我们从技术演变的角度来思考解决方案，如果我们往深层次地去想，不论是el表达式还是jstl标签，其原理都是有一套对应的解释器来解释翻译成java代码，所以，上述的方案是可行的也是有根有据的。且看下面的Demo：


后台:


Book book = new book();// Book.java 这个bean记得要复写toString()方法。
request.setAttribute(“book”,book);

前端页面:


<%
Book book = request.getAttribute(“book”);
out.write(book);
%>

或者

<%=request.getAttribute(“book”)%>

或者

${book}

看完上述的传统的测试方法后，我们现在来看看使用框架是如何测试的？

准备工作：

struts版本：2.3

开发步骤：

  1. 下载组件，引入jar包：阿帕奇组织官网（apache） 下载8个jar包

  2. 配置web.xml

    <!-- 配置Struts核心过滤器 -->
<filter>
    <filter-name>struts2</filter-name>
<filter-class>org.apache.struts2.dispatcher.ng.filter.StrutsPrepareAndExecuteFilter</filter-class>
</filter>

<filter-mapping>
    <filter-name>struts2</filter-name>
    <url-pattern>/*</url-pattern>
</filter-mapping>

  3. 写Action

    // 写Action
public class HelloAction{

    // 处理请求代码！
    public String execute() {
        System.out.println("HelloAction.execute()");
        Book book = new book();// Book.java 这个bean记得要复写toString()方法。
        ServletActionContext.getRequest().setAttribute("book",book);
        return "success";
    }
    public String add() {
        System.out.println("HelloAction.add()");
        return "add";
    }    
}

  4. 配置Action
    配置路径：src/struts.xml

    <?xml version="1.0" encoding="UTF-8" ?>
<!DOCTYPE struts PUBLIC
    "-//Apache Software Foundation//DTD Struts Configuration 2.3//EN"
    "http://struts.apache.org/dtds/struts-2.3.dtd">

<struts>
<package name="test1234" extends="struts-default">

<!-- action: 配置访问路径与处理action类的映射关系！ -->
<!-- 访问路径： http://localhost:8080/TestDemo/add -->
<action name="hello" class="cn.dantefung.action.HelloAction" method="execute">
    <result name="success">/index.jsp</result>
</action>

<!-- 访问路径：http协议+ip地址+端口号+项目名+action名 -->
<!-- http://localhost:8080/TestDemo/add -->
<!-- http://localhost:8080/TestDemo/add.action -->
<action name="add" class="cn.dantefung.action.HelloAction" method="add">
    <result name="add">/add.jsp</result>
</action>

##项目可改进的地方


应该面向接口编程，推荐的项目目录规划如下：

三层架构+MVC规范下的目录结构：
com.dantefung.项目名称.功能/层次

com.dantefung.studentmgr.entity         javabean模型
com.dantefung.studentmgr.dao            dao接口
com.dantefung.studentmgr.dao.impl   dao实现类
com.dantefung.studentmgr.service      service接口
com.dantefung.studentmgr.service.impl  service实现类
com.dantefung.studnetmgr.web.servlet     表现层的servlet类
com.dantefung.studentmgr.web.action     struts2框架的action类
com.dantefung.studentmgr.test         单元测试类 （Junit）
com.dantefung.studentmgr.util         工具类
com.dantefung.studentmgr.exception   自定义异常类


考虑到安全性问题，建议把页面都放在WEB-INF下。然后通过控制器转发到相应的页面。
